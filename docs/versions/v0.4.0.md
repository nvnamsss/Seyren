# v0.4.0

## Purpose
Standardize the three structural layers of SeyrenLabs that have accumulated inconsistencies across the v0.1–v0.3 iteration cycles:

1. **UI Definition** — enforce a consistent UIToolkit/UXML authoring convention across all panels and scenes.
2. **Game ↔ Seyren Protocol** — define a formal boundary contract between scene controllers and the Seyren universe.
3. **Experiment Scene Structure** — bring all five experiment scenes to a common Setup/Controller/ViewController pattern.

No new gameplay features. This version is purely structural/organizational.

- `grep -r "Prefab/Components"` (singular) returns zero results in `Assets/`.
- No `VisualElement` subclass shares a `.cs` file with another named VisualElement subclass (Exception: `TheUILabViewController.cs` carry-over documented).

- No controller accesses `Main.Instance.universe` — all use the injected reference from `ISceneController.Initialize()`.
- `AbilitySystem.OnAbilityCast` fires on every `CastAbility()` call.
- Build passes: zero compile errors, zero CS0414/CS0219 in touched files.
