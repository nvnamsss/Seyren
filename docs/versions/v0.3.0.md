# v0.3.0

## Purpose
Begin isolating Unity-specific dependencies to move toward an engine-agnostic framework (roadmap Phase 1/3), resolve v0.2.0 carry-overs, and add Labs-based visualization of the Damage engine.

Three pillars:
- **Engine Abstraction** — primary focus; strip Unity hard-dependencies from core runtime systems
- **Carry-Overs** — Projectile explosion VFX, Knockup visual effects and target restriction
- **Damage Engine Visualization** — Labs demo for the full damage pipeline

## In Scope
- Engine abstraction for Universe, Action pipeline, and State management runtime systems.
- Adapter extraction for any Unity-specific bindings found in those systems.
- Projectile explosion VFX in Labs.
- Knockup visual arc effect and action restriction in the Seyren Knockup system.
- Damage engine Labs visualization: scene setup, controller, floating text, UI readout.
- On-hit effect proc system demo: `ProcOnHitEffect`, chance-based ability trigger, multi-effect support, Dashboard feed row.
- Architecture documentation update reflecting abstraction progress.

## Out of Scope
- Multi-language IDL and code generators (roadmap Phase 2) — deferred to v0.4.0+.
- Full Unreal / Godot engine adapter implementation.
- Gameplay balance tuning.
- Production-level UI polish and final art direction.
- Broad performance optimization not required for visualization goals.
- Feature expansion beyond Damage, Knockup, and Projectile VFX.
