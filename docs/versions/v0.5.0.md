# v0.5.0

## Purpose
Introduce the **AbilityGraph System** — a runtime execution graph framework for defining combat abilities.

Every ability becomes a directed graph of `IAbilityNode`s ticked each server frame. Casting an ability creates an `AbilityGraphInstance` that drives the graph's entry node. Composite nodes control execution flow; leaf nodes produce gameplay effects; event nodes react to runtime signals; subgraph nodes spawn independent child instances; modifiers transform graph definitions at cast time.

The system integrates into the existing `Ability.TickEffect()` chain without changes to `AbilitySystem` or `LabUniverse`.

## Architecture Boundaries

Graph Ability System handles:
- Ability execution, projectiles, damage, buffs/debuffs, summons, timers, triggers, ability modifiers

Graph Ability System does NOT handle:
- Physics simulation, AI decision making, pathfinding, economy, world simulation
- Those are accessed through node delegates that call into existing systems (DamageEngine, ISpace, etc.)

---

## Key Design Decisions

| Decision | Rationale |
|---|---|
| `AbilitySystem` unchanged | `GraphAbility.TickEffect()` manages its own instance list — zero changes to existing pipeline |
| Context event bus on `AbilityGraphContext` | No global bus; events scoped to each cast instance |
| Modifiers applied at `Cast()` time | Instances are immutable post-creation; graph transformation is pre-instantiation only |
| `SubgraphNode` shares parent context | Enables variable sharing between parent/child graphs; caller clones context if isolation needed |
| Demo lives in `Assets/Labs/`, not Seyren | Keeps the framework clean; demo depends on Unity/MonoBehaviour specifics |
